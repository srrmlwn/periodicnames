# Phase 1 Task Breakdown

Cartoonish UI + Slick Animations. Generated by codebase analysis.

## Status Summary (baseline)

Already done:
- Animation phase state machine (input → processing → results) in App.tsx
- Refresh button
- Per-category colors in `colorSchemes.ts`
- `element-fade-in`, `element-glow`, `element-pulse` keyframes in index.css (but duplicated — bug)
- Staggered animation in ResultDisplay via setTimeout accumulation

Known bugs in current code:
- `index.css` defines `.element-glow` and `.element-pulse` twice; second definition silently wins
- Periodic table fades to `opacity-0` during `processing` and stays hidden during `results` — so highlight animations never show
- ElementTile uses hardcoded `w-6 h-6` but ResultDisplay wraps tiles in `w-14 h-14` — tiles don't fill their container
- `fakeElements.find()` always picks the first variant, making the random fun-name selection unreachable
- Stagger delay is 100ms/tile; target is 50–80ms

  _Effort: XS_

- [ ] **Replace greedy matcher with DP** (`src/utils/elementMatcher.ts`)
  Current greedy fails to guarantee maximum real element count for longer names.
  DP approach: `dp[i]` = max real elements from position `i` to end. Process right-to-left. At each `i`, try lengths 1 and 2:
  - If `substring(i, i+len)` is a real element → candidate = `dp[i+len] + 1`
  - If length 1 falls back to fake → candidate = `dp[i+1] + 0`
  Pick max candidate, record choice in `choice[i]`. Reconstruct forward.
  Handle spaces by splitting into words, running DP per word.
  _Effort: M_

## Notes on Key Implementation Choices

**Staggered delay without Framer Motion**: Keep the existing `setTimeout` accumulation pattern in ResultDisplay. Pair it with `tilePop` keyframe + `cubic-bezier(0.34, 1.56, 0.64, 1)` for a spring-like overshoot feel. No library needed.

**CSS shimmer (no canvas)**: `background-size: 200%` makes the gradient wider than the tile. Animating `background-position` sweeps the bright streak across. Works with pure CSS, no JS.

**`w-full h-full` in ElementTile**: Add a `size` prop to switch text sizes between the periodic table (tiny) and result display (readable). Both use the same component, just sized by their container.

**DP matcher**: The reconstruction phase needs a `choice[i]` array storing which length was selected at each position. Space handling: split input on spaces, run DP on each word segment, then interleave with space separators in the output array.

**Exit animation race condition**: Store the exit `setTimeout` id in a `useRef` and `clearTimeout` it on new submit. Otherwise rapid re-submission causes the old exit timer to fire mid-new-animation.
